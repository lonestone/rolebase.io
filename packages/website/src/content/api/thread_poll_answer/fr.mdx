## Relations

### Relations objet

- `activity` — L'activité de discussion (sondage) à laquelle cette réponse appartient

## Exemples de requêtes

### Lister les réponses d'un sondage

```graphql
query GetPollAnswers($activityId: uuid!) {
  thread_poll_answer(where: { activityId: { _eq: $activityId } }) {
    id
    activityId
    userId
    choicesPoints
    createdAt
  }
}
```


## Permissions

L'accès aux réponses de sondage est contrôlé en fonction des permissions de la discussion et de l'activité :

- Les **utilisateurs** peuvent consulter, soumettre, modifier et supprimer leurs propres réponses dans les discussions accessibles
- Les **propriétaires de discussion** peuvent supprimer toutes les réponses de sondage de leurs discussions
- Les **administrateurs et propriétaires de l'organisation** peuvent supprimer toutes les réponses de sondage des discussions de leur organisation

<Callout type="info">
Chaque utilisateur ne peut soumettre qu'une seule réponse par sondage. Le champ `choicesPoints` est un tableau d'entiers où chaque élément représente les points attribués au choix de sondage correspondant. Les réponses de sondage peuvent être utilisées pour la prise de décision, la planification, la collecte d'avis et les enquêtes rapides.
</Callout>
