## Relationships

### Object Relationships

- `bucket` ‚Äî The storage bucket containing the file

### Array Relationships

- `members` ‚Äî Members using this file (e.g., as a profile picture)
- `orgs` ‚Äî Organizations associated with this file

## Storage Buckets

Files are stored in buckets that define storage parameters:

- `cacheControl` ‚Äî Cache control settings
- `downloadExpiration` ‚Äî Download link expiration time
- `maxUploadFileSize` ‚Äî Maximum allowed file size
- `minUploadFileSize` ‚Äî Minimum allowed file size
- `presignedUrlsEnabled` ‚Äî Whether presigned URLs are enabled for secure temporary access

## Query Examples

### Get a Specific File

```graphql
query GetFile($id: uuid!) {
  file_by_pk(id: $id) {
    id
    name
    mimeType
    size
    createdAt
    metadata
  }
}
```

## Permissions

File access is controlled by several factors:

- **Organization membership** ‚Äî Members can access files associated with their organization
- **Member role** ‚Äî Upload and modification rights depend on the member's role
- **Bucket permissions** ‚Äî Storage bucket configuration governs access and size limits
- **Uploader rights** ‚Äî File metadata can be updated by the original uploader
- **Virus scanning** ‚Äî Uploaded files are scanned for security

<Callout type="info">
Files are stored securely in configurable storage buckets. Presigned URLs enable secure temporary access, and MIME type validation ensures proper file handling. ETags support caching and conditional requests for efficient file delivery.
</Callout>
