## Overview

Threads are Rolebase's tool for **asynchronous discussions**. They provide a structured space for conversations that do not need to happen in real time, such as governance proposals, project updates, feedback requests, and more.

Each thread belongs to a role and progresses through defined statuses.

## Creating a Thread

1. Navigate to a role's threads section.
2. Click **New Thread**.
3. Enter a title and opening message.
4. The thread is created in **Preparation** status by default.

## Thread Statuses

Threads move through four statuses:

| Status | Description |
|---|---|
| **Preparation** | The thread is being drafted. Only the creator can see it until it is activated. |
| **Active** | The thread is open for discussion. All role members can participate. |
| **Blocked** | The discussion is paused, waiting for external input or a decision. |
| **Closed** | The discussion is concluded. The thread is archived for reference. |

## Activities and Messages

Thread activities are the individual messages within a thread. Members can:

- Post text messages with rich formatting.
- Reply to specific messages.
- Add **reactions** to messages (emoji reactions for quick feedback).

Discussions are threaded, keeping related messages grouped together.

## Polls

Threads support **polls** for structured decision-making. When you need input from the group, create a poll within a thread.

### Poll Options

| Option | Description |
|---|---|
| **Multiple Choice** | Participants select one or more options from a predefined list |
| **Points** | Participants distribute a set number of points across options to indicate preference strength |
| **Anonymous** | Votes are hidden. Results are shown without revealing who voted for what |
| **Randomize** | The order of options is randomized for each voter to reduce order bias |

<Callout type="tip" title="Consent-Based Decisions">
  Polls are a great tool for consent-based decision-making in Sociocracy. Create a poll with options like "Consent", "Object", and "Stand Aside" to formalize rounds.
</Callout>

### Creating a Poll

1. Open a thread.
2. Click the **Poll** button in the message composer.
3. Add a question and define the options.
4. Configure settings (multiple choice, points, anonymous, randomize).
5. Publish the poll.

Participants see the poll in the thread and can cast their votes. Results update in real time.

## Thread Privacy

Threads can be marked as **private**. Private threads are only visible to the thread creator and explicitly added participants. They do not appear in the role's thread list for other members.

## Extra Members

By default, a thread is visible to all members of its role. You can add **extra members** from other roles to participate in a thread without giving them full role membership.

This is useful for:

- Cross-role collaboration on specific topics.
- Getting input from subject-matter experts in other parts of the organization.
- Involving stakeholders who do not need ongoing role membership.

<Callout type="info" title="Notifications">
  Extra members receive notifications for the thread just like regular role members.
</Callout>

## Threads in Meetings

Active threads are reviewed during the **Threads** step of role meetings. This ensures that asynchronous discussions are regularly surfaced in synchronous settings, keeping topics from stalling.
